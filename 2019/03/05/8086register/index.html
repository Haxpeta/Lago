x86 8086 寄存器

2019-03-04

文章目录

  1. 通用寄存器(GENERAL PURPOSE REGISTERS)
  2. 段寄存器(SEGMENT REGISTERS)
  3. 特殊目的寄存器(SPECIAL PURPOSE REGISTERS)
  4. flag 寄存器(Program Status Word)

基本 8086 寄存器以及部分用途列举.

8086cpu 的寄存器共有AX,BX,CX,DX,SI,DI,SP,BP,IP,CS,SS,DS,ES,PSW14 个寄存器.

通用寄存器(GENERAL PURPOSE REGISTERS)

这 14 个寄存器中, 有 8 个通用寄存器, 分别是AX,BX,CX,DX,SI, DI, BP, SP 它们分别用于存放一般通用型数据.

其中 AX,BX,CX,DX通用寄存器又分为高位寄存器与低位寄存器, 分别表示为AH与AL. B,C,D也可以作同样的区分. 但是其它四个不行.

这几个寄存器的英文名字以及作用为:

  1. AX

    累加寄存器(the accumulator register)

    • 产生简短的机器代码(Generates shortest machine code)
    • 算数, 逻辑和数据传输(Arithmetic, logic and data transfer)
    • One number must be in AL or AX
    • 乘法和除法(Multiplication & Division)
    • 输入输出(Input & Output)
  2. BX

    基址寄存器(the base address register)

  3. CX

    计数寄存器(the count register)

    • 使用LOOP等指令的时候调用(Iterative code segments using the LOOP instruction)
    • 重复的执行某个指令(Repetitive operations on strings with the REP command)
    • Count (in CL) of bits to shift and rotate
  4. DX

    数据寄存器(the data register)

    • DX和AX寄存器相连接来存储 32 位数字供DIV和MUL指令使用(DX:AX concatenated into 32-bit register for some MUL and DIV operations)
    • 使用IN和OUT的时候指定端口(Specifying ports in some IN and OUT operations)
  5. SI

    源地址索引寄存器(source index register)

    • 可以被用作数据指针(Can be used for pointer addressing of data)
    • 在某些字串处理指令中充当源(Used as source in some string processing instructions)
    • 相对于DS寄存器的偏移地址(Offset address relative to DS)

  6. DI

    目的地址索引寄存器(destination index register)

    • 可以被用作数据指针(Can be used for pointer addressing of data)
    • 在某些字串处理指令中充当目标(Used as destination in some string processing instructions)
    • 相对于ES寄存器的偏移地址(Offset address relative to ES)

  7. BP

    基址指针寄存器(base pointer)

    • 主要用于访问存储在栈中的变量(Primarily used to access parameters passed via the stack )
    • 相对于SS寄存器的偏移地址(Offset address relative to SS)
  8. SP

    栈指针寄存器(stack pointer)

    • 总是指向栈顶(Always points to top item on the stack)
    • 相对于SS寄存器的偏移地址(Offset address relative to SS)
    • 操作的时候总是以字为单位(Always points to word)
    • 一个空栈会有SP=FFFEH(An empty stack will had SP = FFFEh)

段寄存器(SEGMENT REGISTERS)

8086 有 4 个段寄存器: CS, DS, SS, ES.

段寄存器如其名所示, 是用来存放段地址的寄存器, 为 16 位. 同时,仅有一个段寄存器是不够的, 还需要配合一个可以提供偏移地址的立即数或者寄存器来使用.

  • CS

    指令段寄存器(code segment)

  • DS

    数据段寄存器(data segment)

  • ES

    额外的段寄存器(extra segment register)

    • 它的功能由程序员决定
  • SS

    栈段寄存器(stack segment)

默认状态下, BX, SI 和 DI 寄存器与DS段寄存器一起工作. BP 和 SP 寄存器与 SS 段寄存器一起工作.

特殊目的寄存器(SPECIAL PURPOSE REGISTERS)

这里面就一个IP寄存器, 总是和CS寄存器一起工作, 并且, IP总是指向下一个需要执行的指令地址.

flag 寄存器(Program Status Word)

作用:

  1. 存储相关命令的执行结果
  2. 为 cpu 指令的某些行为提供依据
  3. 控制 cpu 的相关工作方式

8086 中的标志寄存器(flag)为 16 位, 存储的内容为程序状态字(PSW).

其中, flag 寄存器和通用寄存器不同, 它是按照每个特殊的位工作的. 每一个位置上的位都具有特殊的含义.

8086 中,0, 2,4,6,7,8,9,10,11位有特殊的含义.

0 2 4 6 7 8 9 10 11
CF PF AF ZF SF TF IF DF OF
  • CF

    进位标志 (Carry Flag)

    • 当有一个无符号数字溢出时, 设为1, 否则为0.
  • PF

    奇偶标志位 (Parity Flag)

    • 记录相关指令的结果中1的比特位个数是否为偶数, 如果是为1,否则0.
    • 如果结果大小是一个字, 只有低 8 位会被计算!
    • 其中0个1也是偶数个1
  • AF

    辅助进位标志 (Auxiliary Flag)

    • 在字操作时, 发生低字节向高字节进位或借位时, 设置为1, 否则为0, 在字节操作时则相应的低 4 位向高 4 位进位或者借位.
  • ZF

    零标志位 (Zero Flag)

    • 记录相关指令的执行结果是否为 0, 如果为 0 , 那么此位为1, 否则0.
  • SF

    符号标志位 (sign Flag)

    • 记录相关指令的执行结果是否为负, 若是为负, 则为1, 否则0.
    • 需要注意的是, 在相关的指令中, 起始默认已经包含了两种结果, 而如何看待执行后的结果. 将它看成符号数还是无符号数, 由我们自己决定.
  • TF

    内中断标志位 (Trap Flag)

    • 如果为1, 那么在执行完当前的指令后, 执行单步中断.
  • IF

    外中断标志位(Interrupt enable Flag)

    • 如果为1, 那么响应可屏蔽外中断.
  • DF

    方向标志位 (Direction Flag)

    • 用作某些指令的执行方向, 如果设为0, 那么正向, 如果为1, 反向.
  • OF

    溢出标志位 (Overflow Flag)

    • 记录了相关命令的有符号数字执行结果中, 是否溢出. 溢出则为1,否则0.
